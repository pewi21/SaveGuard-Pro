Core Philosophy

Two Tools in One

SaveGuard Pro divides its responsibilities into two independent systems to perfectly match your creative pipeline.

1. AutoSave

Fast, frequent saves for live-preview workflows.

If you work alongside a 3D application, a game engine, a web browser, or any tool that watches a file for changes, AutoSave continuously writes your Photoshop document so external tools always show the latest version.

  • Near-live connection to your pipeline
  • No more manual Ctrl/Cmd+S
  • Set short intervals without interruptions

2. AutoBackup

Timestamped safety copies for long editing sessions.

Creates dated backup files at a separate interval, independently of AutoSave. Even if you overwrite something by accident or a file becomes corrupted, you always have a recent copy to fall back to.

  • Set a maximum copy count
  • Set a maximum folder size
  • Automatically removes oldest backups to save disk space

Smart By Design

Engineered for reliability

A complete safety net that stays out of your way and adapts to how you work.

🧠

Smart Delay

Monitors actual save duration and automatically raises the check interval when saves take too long, keeping Photoshop responsive on heavy files. Acts as a minimum safe floor.

🎯

Cohesion Delay

Waits for a pause in your editing activity before triggering a save. Saves never fire mid-stroke or during rapid changes, preserving your flow.

👁️

Intelligent Change Detection

In history-tracking mode, SaveGuard scans Photoshop history states to detect real edits. Routine actions never trigger unnecessary saves.

🚫

Ignored Operations

Define named lists of history states to skip (like "Lasso" or "Move Selection"). Assign different lists to AutoSave and AutoBackup independently.

📁

Smart Format Handling

Full overwrite support for PSD, PSB and TIFF. Safely handles PNG and JPG by automatically switching to 'Save As' for multi-layer files. Smart alerts warn you about formats needing manual flattening or blocked by API limits.

🔒

Crash-safe & Reliable

Settings are written atomically using .tmp/.bak recovery. Includes automatic retry (up to 3 times) with per-error-code delays for locked files or slow disks.
